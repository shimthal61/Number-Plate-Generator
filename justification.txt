Justification & Sources
src/ layout
The src/ layout is the current PyPA (Python Packaging Authority) recommendation. It prevents the package from being accidentally importable from the project root without installing it, which catches import path bugs early.

Source: packaging.python.org — src layout (updated 2024)
Source: Hynek Schlawack, "Testing & Packaging" (2022, still canonical): hynek.me/articles/testing-packaging
pyproject.toml (no setup.py / requirements.txt)
PEP 517 (2015) and PEP 621 (2020) standardised pyproject.toml as the single source of truth for project metadata, build config, and tool config (pytest, coverage, etc.). setup.py is legacy.

Source: PEP 621 — project metadata in pyproject.toml
Source: packaging.python.org — Writing your pyproject.toml (2024)
tests/ at root with conftest.py
Keeping tests outside src/ is the pytest-recommended convention. conftest.py is pytest's standard fixture/configuration file.

Source: docs.pytest.org — Good Integration Practices (2024)
No requirements.txt
For a self-contained package, dependencies live in pyproject.toml under [project.dependencies]. requirements.txt is for deployment pinning, not package definition.

Source: pip.pypa.io — Requirements Files vs pyproject.toml (2024)
